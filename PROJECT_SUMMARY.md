## 🔒 Безопасность и конфиденциальность

### API ключ
- ✅ Хранится только в `chrome.storage.sync`
- ✅ Не передается в content-скрипты
- ✅ Не логируется в консоль
- ✅ Используется только Service Worker'ом

### Пользовательские данные
- ✅ Текс отправляется только в OpenRouter API
- ✅ Никакие данные не хранятся на внешних серверах
- ✅ Все предварительно обработанные данные кэшируются локально

### Браузерные API
- ✅ `chrome.storage.sync` - для синхронизированных настроек
- ✅ `chrome.storage.local` - для локального кэша
- ✅ `chrome.runtime.sendMessage()` - для межпроцессной связи
- ✅ `chrome.tabs.sendMessage()` - для отправки в content-скрипты

## 🎯 Производительность

### Оптимизации
1. **Дебаунс (600ms)** - не отправлять на каждый символ
2. **Кэширование** - результаты хранятся в памяти
3. **Ленивая инициализация** - поля инициализируются по мере появления
4. **MutationObserver** - отслеживает новые элементы
5. **Ограничение размера** - макс 4000 символов за запрос

### Метрики
- Время первого повтора проверки: ~100ms (дебаунс)
- Время отправки запроса: ~50ms
- Время получения ответа: 1-3 секунды (зависит от модели)
- Время рисования ошибок: ~10ms
- Требуемая память: ~10-20 MB
