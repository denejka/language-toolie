## 🔴 Частые ошибки и решения

### Ошибка 1: "Картофель картофельный"

**Проблема:** AI находит ошибки, которых нет (особенно в творческих текстах)

**Пример:**
```
❌ Тридцать три Алтайских гор
   ↑ "Алтайских" - ошибка? Но это название!
```

**Решение:**
```javascript
// Создать пользовательский промпт
customPrompt: `
  Пропускай:
  - Собственные имена (города, имена люей)
  - Названия организаций
  - Контекстуальные повторы (если они намеренные)
  
  Текст: {{TEXT}}
`
```

### Ошибка 2: "В чем разница между 'что' и 'чего'?"

**Проблема:** Разные модели дают разные рекомендации для одного текста

**Пример:**
```
Текст: "Что ты имеешь в виду?"

Модель 1: ✓ Правильно
Модель 2: ✗ Неправильно, нужно "Чего ты имеешь в виду?"
```

**Решение:**
```javascript
// Установить предпочтительный стиль
customPrompt: `
  Используй правила современного русского языка
  (ГОСТ 2.105-95 и РАН рекомендации).
  
  Разрешено:
  - "Что ты имеешь в виду?" (вариант 1)
  - "Чего ты имеешь в виду?" (вариант 2)
  
  Только укажи, если оба варианта используются неправильно.
  
  Текст: {{TEXT}}
`
```

### Ошибка 3: "Зависание при большом тексте"

**Проблема:** Расширение медленно обрабатывает тексты >2000 символов

**Пример:**
```javascript
// Плохо: в поле 5000+ символов
Ожидание: секунды...
Результат: медленно, API может вернуть ошибку
```

**Решение:**
```javascript
// Хорошо: ограничить размер текста
const maxLength = 3000;
if (text.length > maxLength) {
  // Опция 1: Проверить только первые 3000 символов
  text = text.substring(0, maxLength);
  
  // Опция 2: Показать предупреждение пользователю
  showNotification('⚠️ Текст большой, проверка может быть неполной');
}
```

### Ошибка 4: "JSON парсинг не удается"

**Проблема:** AI вернула неправильный JSON

```javascript
// Bad JSON от AI:
{
  corrections: [
    {
      start: 0,
      end: 5,
      wrong: "текст,    // ЗАБЫЛА ЗАКРЫВАЮЩУЮ КАВЫЧКУ
      suggestions: ["text"],
    }
  ]
}
```

**Решение в background.js:**
```javascript
try {
  // Попытка основного парсинга
  const parsed = JSON.parse(jsonString);
  return parsed.corrections;
} catch (e) {
  // Fallback: Очистить JSON
  jsonString = jsonString
    .replace(/,\s*\]/g, ']')      // Удалить запятые перед ]
    .replace(/,\s*}/g, '}')       // Удалить запятые перед }
    .replace(/'/g, '"');          // Заменить одинарные кавычки на двойные
    
  try {
    const parsed = JSON.parse(jsonString);
    return parsed.corrections || [];
  } catch (e) {
    console.error('JSON очистка не помогла');
    return [];
  }
}
```

### Ошибка 5: "Пустой список моделей"

**Проблема:** При первой загрузке модели не загружаются

```javascript
// Плохо: Прямой запрос без проверки
const response = await fetch('https://openrouter.io/api/v1/models', {
  headers: { 'Authorization': `Bearer ${apiKey}` }
});
// Может вернуть 401 если ключ неправильно
```

**Решение:**
```javascript
// Хорошо: С fallback локальным списком
async function getAvailableModels() {
  try {
    const response = await fetch('https://openrouter.io/api/v1/models', {
      headers: { 'Authorization': `Bearer ${apiKey}` }
    });
    
    if (!response.ok) {
      throw new Error(response.statusText);
    }
    
    const data = await response.json();
    return data.data.filter(m => m.id.includes(':free'));
  } catch (error) {
    console.warn('Ошибка загрузки моделей:', error);
    // Fallback к встроенному списку
    return DEFAULT_FREE_MODELS;
  }
}
```
