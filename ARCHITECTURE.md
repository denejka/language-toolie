# 🏗️ Архитектура AI Grammar & Style Checker

## Обзор системы

Расширение использует **Manifest V3** архитектуру Chrome с разделением на три основные части:

```
┌─────────────────────────────────────────────────────────────┐
│                     Chrome Browser                           │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  Content Script (content.js)          [На каждой странице]
│  │  ├─ Сканирование текстовых полей                        │ │
│  │  ├─ Отслеживание ввода текста                           │ │
│  │  ├─ Визуализация ошибок (волнистые линии)              │ │
│  │  ├─ Всплывающие окна исправлений                        │ │
│  │  └─ Применение исправлений                             │ │
│  └─────────────────────────────────────────────────────────┘ │
│                            ▲                                   │
│             chrome.runtime.sendMessage()                      │
│                            │                                   │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  Service Worker (background.js)    [Один на всё браузер]
│  │  ├─ Слушание сообщений от content-скриптов             │ │
│  │  ├─ Обработка API запросов к OpenRouter                │ │
│  │  ├─ Парсинг JSON ответов                              │ │
│  │  ├─ Управление ошибками и повторами                   │ │
│  │  └─ Хранение настроек (chrome.storage)                │ │
│  └─────────────────────────────────────────────────────────┘ │
│                            ▲                                   │
│                  HTTPS запросы                                 │
│                            │                                   │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  OpenRouter API                 (https://openrouter.io)|
│  │  ├─ POST /v1/chat/completions (проверка текста)       │ │
│  │  └─ GET /v1/models (список доступных моделей)        │ │
│  └─────────────────────────────────────────────────────────┘ │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  Popup (popup.html/js)              [При клике на иконку]
│  │  ├─ Быстрый статус расширения                          │ │
│  │  ├─ Переключение включения/отключения                 │ │
│  │  ├─ Очистка кэша                                       │ │
│  │  └─ Быстрая ссылка на настройки                       │ │
│  └─────────────────────────────────────────────────────────┘ │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  Options (options.html/js)      [Страница настроек]    │ │
│  │  ├─ Управление API ключом                              │ │
│  │  ├─ Выбор модели                                       │ │
│  │  ├─ Настройка уровня проверки                         │ │
│  │  ├─ Исключения и белые списки                         │ │
│  │  └─ Пользовательское промптирование                   │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

## 🔐 Безопасность данных

### Хранение API ключа

```javascript
// Только в chrome.storage.sync (зашифровано Chrome)
{
  apiKey: 'sk-or-...',  // Никогда не отправляется в content-скрипты
  // ... другие настройки
}
```

**Почему это безопасно:**
- Service Worker имеет доступ только к API ключу
- Content Script может отправлять только текст и жанр
- Service Worker отправляет запрос напрямую к OpenRouter
- Ключ не виден в console на сайтах

### Изоляция контекста

```javascript
// Content Script не может получить доступ к:
- Памяти Service Worker'а
- chrome.storage.sync данным (зашифрованным)
- Другим расширениям

// Service Worker не может:
- Взаимодействовать с DOM страницы
- Видеть введенные пользователем данные напрямую
- Отправлять данные куда-либо, кроме OpenRouter API
```

## 📊 Обработка ошибок

### Цепочка fallback'ов

```javascript
// 1. Основная попытка
//    ↓ Ошибка
// 2. Повторный запрос с более строгим промптом
//    ↓ Ошибка при парсинге JSON
// 3. Попытка парсить вручную и извлечь JSON
//    ↓ Ошибка
// 4. Вернуть пустой массив исправлений
//    ↓
// Пользователь видит уведомление об ошибке
```

### Обработка исключений

```javascript

  // Отправка запроса к OpenRouter

  const content = data.choices[0].message.content;
  
  // Парсинг ответа от AI
  const parsed = JSON.parse(content);
```

## 🎯 Производительность

### Оптимизации

1. **Дебаунс (600ms)**
   - Не отправлять запрос на каждый символ
   - Ждем 600ms после последнего ввода

2. **Кэширование**
   - Результаты хранятся в памяти (`currentCorrections`)
   - Повторные открытия поля показывают старые результаты

3. **Ленивая инициализация**
   - Content Script инициализируется один раз
   - MutationObserver отслеживает новые поля

4. **Ограничение размера текста**
   - Максимум 4000 символов за запрос
   - Большой текст разбивается на чанки (опционально)

## 🌐 Интеграция с OpenRouter API

### Используемые endpoints

| Endpoint | Метод | Параметры |
|----------|-------|-----------|
| `/chat/completions` | POST | model, messages, temperature, max_tokens |
| `/models` | GET | Без параметров |

### Формат запроса

```javascript
{
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: {
    model: 'google/gemini-2.0-flash-exp:free',
    messages: [
      { role: 'system', content: '...' },
      { role: 'user', content: '...' }
    ],
    temperature: 0.3,  // Низкая для консистентности
    max_tokens: 4000
  }
}
```

### Формат ответа

```json
{
  "choices": [
    {
      "message": {
        "content": "{\"corrections\": [{...}]}"
      }
    }
  ]
}
```
