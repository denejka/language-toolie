# 📦 Инструкция по установке и использованию

## Предварительные требования

- **Chrome версии 96+** (или другой браузер на Chromium)
- **Аккаунт на OpenRouter.io**
- **API ключ от OpenRouter** с активным балансом

## 🔑 Шаг 1: Получение API ключа

### 1.1 Регистрация на OpenRouter

1. Перейти на сайт [OpenRouter.io](https://openrouter.io)
2. Нажать кнопку "Sign Up" в правом верхнем углу
3. Зарегистрироваться с помощью Google, GitHub или Email

### 1.2 Активирование API ключа

1. Логирование: вход в личный кабинет
2. Перейти в раздел **Settings** → **API Keys**
3. Нажать кнопку **"Create New Key"**
4. Скопировать ключ (начинается с `sk-or-`)
5. **Сохранить его в безопасном месте** (пароль используется только один раз!)

### 1.3 Пополнение баланса (опционально)

Для использования **бесплатных моделей** (`:free`) пополнение не требуется!  
Однако можно добавить пару долларов для большей гибкости:

1. В личном кабинете → **Billing** → **Add Credits**
2. Выбрать сумму (минимум $5)
3. Оплатить через карту или другой способ

## 💻 Шаг 2: Установка расширения

### 2.1 Загрузка файлов

1. **Вариант A: Git**
   ```bash
   git clone https://github.com/denejka/language-toolie.git
   cd language-toolie
   ```

2. **Вариант B: Как ZIP**
   - Загрузить ZIP файл
   - Распаковать в удобное место

### 2.2 Загрузка в Chrome

1. Открыть Chrome
2. Перейти на страницу расширений: `chrome://extensions/`
3. **Включить "Режим разработчика"** (переключатель в правом верхнем углу)
4. Нажать кнопку **"Загрузить распакованное расширение"**
5. Выбрать папку `/language-toolie`
6. Расширение появится в списке активных расширений

### 2.3 Проверка установки

- Иконка расширения должна появиться в тулбаре браузера (верхний правый угол)
- Если иконки нет, можно зафиксировать расширение:
  - Нажать кнопку расширений (собираются пазлом)
  - Найти "AI Grammar & Style Checker"
  - Нажать на иконку булавки

## ⚙️ Шаг 3: Первая настройка

### 3.1 Открытие настроек

1. Нажать на иконку расширения в тулбаре
2. В меню нажать кнопку **"⚙️ Настройки"**
3. Откроется полная страница настроек

### 3.2 Вкладка "Аккаунт"

**API Key:**
- Вставить скопированный ключ OpenRouter
- Он будет замаскирован (видны только последние 4 символа)
- Нажать кнопку **"Проверить соединение"**
- Должно вывести сообщение "✓ Соединение успешно"

**Выбор модели:**
- Нажать кнопку **"🔄 Обновить список моделей"**
- Выбрать понравившуюся модель:
  - **Google Gemini 2.0** - самая быстрая ✨
  - **Llama 3.3 70B** - профессиональный уровень
  - **DeepSeek R1** - глубокий анализ
  - **Mistral 7B** - легкая версия
  - **Qwen 2 7B** - хорошо работает на русском

### 3.3 Вкладка "Проверка"

**Уровень глубины проверки:**
- **1** = Только орфография (быстро)
- **2** = Грамматика и пунктуация (рекомендуется)
- **3** = Полный анализ + логика (полно)

**Язык интерфейса:**
- Выбрать Русский или English

**Жанр текста:**
- General (обычный текст)
- Academic (деловая переписка, эссе)
- Creative (творческие писания)

### 3.4 Вкладка "Исключения"

**Включение/Отключение расширения:**
- Переключатель вверху страницы
- Расширение можно временно отключить

**Сайты без проверки:**
- Добавить домены, где расширение не будет работать
- Пример: `gmail.com`, `facebook.com`

## 🚀 Использование расширения

### Первое использование

1. Открыть любую веб-страницу с текстовым полем:
   - Текстовое поле в поиске Google
   - Email в Gmail
   - Комментарий на YouTube или Reddit
   - TextArea в любом сервисе

2. Начать печатать текст с ошибкой:
   ```
   Этот текст сожержит оишбку.
   ```

3. После 600 мс текст будет отправлен на проверку
4. Под текстом появится **красное волнистое подчеркивание**
5. Нажать на подчеркивание → появится всплывающее окно
6. Выбрать вариант исправления или нажать "Исправить это"

### Всплывающее окно исправления

**Содержит:**
- Название ошибки и неправильный текст
- Объяснение, почему это неправильно
- Варианты исправления (нажать для быстрого исправления)
- Кнопки внизу:
  - "Исправить это" - применить первый вариант
  - "Исправить всё" - применить все правки в поле сразу

### Быстрое меню (Popup)

Нажать на иконку расширения → появится меню с:
- **Статус расширения** - включено/отключено
- **Количество проверленных полей** на текущей странице
- **Текущая модель и уровень проверки**
- **Кнопки:**
  - Включить/Отключить проверку
  - 🗑 Очистить кэш
  - ⚙️ Открыть полные настройки
  - ❓ Справка и информация

## 🔧 Расширенные функции

### Пользовательский промпт

Для специализированных проверок можно задать собственный промпт:

1. Вкладка "Проверка" → Раздел "Пользовательский промпт"
2. Написать свои инструкции для ИИ
3. Использовать `{{TEXT}}` для подстановки текста

**Пример промпта для стартапов:**
```
Ты — редактор для стартапа. Проверь текст на:
- Профессиональный тон
- Ясность изложения
- Отсутствие бюрократизма

Ответ в JSON...
Текст: {{TEXT}}
```

### Исключаемые проверки

Во вкладке "Исключения" можно отметить:
- ☑️ Не проверять CAPS LOCK
- ☑️ Не проверять URL ссылки
- ☑️ Не проверять код (в backticks)
- ☑️ Не проверять email адреса

### Параметры производительности

В разделе "Расширенные параметры":
- **Минимальная длина текста** - сколько символов нужно, чтобы начать проверку
- **Задержка перед отправкой** - дебаунс в миллисекундах (600 мс рекомендуется)
- **Максимум символов** - лимит за раз (4000 рекомендуется)

## 🐛 Диагностика и решение проблем

### Проблема: "Не удалось установить соединение"

**Решение:**
1. Проверить, что API ключ введен правильно (без пробелов)
2. Убедиться, что ключ начинается с `sk-or-`
3. Проверить интернет соединение
4. Перейти на OpenRouter.io - проверить, не заблокирован ли аккаунт
5. Попробовать создать новый ключ

### Проблема: "Текст не проверяется"

**Решение:**
1. Убедиться, что расширение активно (иконка в тулбаре)
2. Нажать на расширение → проверить, что статус "Активно"
3. Убедиться, что это `input`, `textarea` или `contenteditable` элемент
4. Открыть F12 → Console → проверить ошибки
5. Попробовать очистить кэш (нажать 🗑 в меню)

### Проблема: "Расширение зависло / медленно работает"

**Решение:**
1. Очистить кэш (меню расширения → 🗑 Чистка)
2. Перезагрузить страницу (Ctrl+R или Cmd+R)
3. Выбрать более быструю модель (например, Gemini вместо Llama)
4. Уменьшить уровень глубины проверки (с 3 на 2)
5. Увеличить задержку дебауса (с 600 на 1000 мс)

### Проблема: "Ошибка парсинга JSON"

**Решение:**
1. Расширение автоматически повторит запрос (не требует действий)
2. Если ошибка повторяется:
   - Попробовать другую модель
   - Уменьшить размер текста
   - Включить более простой режим проверки (режим 1)

### Проблема: "Расширение разбирает случайные ошибки"

**Решение:**
1. Это может зависеть от модели
2. Попробовать другую модель (DeepSeek часто точнее)
3. Установить пользовательский промпт с более строгими инструкциями
4. Добавить исключаемые проверки (например, "Не проверять URL")

## 📊 FAQ (Часто задаваемые вопросы)

**Q: Почему расширение не работает на gmail.com?**  
A: Gmail имеет специальную защиту. Проверка работает в окне редактирования письма.

**Q: Сколько стоит использование?**  
A: Бесплатные модели (`:free`) полностью бесплатны на OpenRouter!

**Q: Где хранится мой API ключ?**  
A: Только в браузере, локально в `chrome.storage.sync`. Никуда не отправляется.

**Q: Можно ли использовать расширение на нескольких устройствах?**  
A: Да! chrome.storage.sync синхронизирует настройки через облако Google.

**Q: Как отключить расширение на определенном сайте?**  
A: Во вкладке "Исключения" добавить домен в "Сайты без проверки".

**Q: Что делать, если забыл API ключ?**  
A: Заново создать на openrouter.io в разделе Settings → API Keys.
