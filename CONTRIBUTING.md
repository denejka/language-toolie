# 🤝 Рекомендации по контрибьюции

Спасибо за интерес к развитию AI Grammar & Style Checker! 🎉

## Как начать разработку

### 1. Форкинг и клонирование

```bash
# Форк репозитория на GitHub
# Затем:

git clone https://github.com/ВАШ-ПРОФИЛЬ/language-toolie.git
cd language-toolie
git remote add upstream https://github.com/denejka/language-toolie.git
```

### 2. Создание ветки

```bash
# Обновить main
git fetch upstream
git rebase upstream/main

# Создать новую ветку для фичи
git checkout -b feature/имя-функции
# Или для баг-фикса
git checkout -b fix/имя-ошибки
```

### 3. Установка и тестирование

```bash
# Загрузить расширение в режиме разработчика
# chrome://extensions/
# → Загрузить распакованное расширение
# → Выбрать папку language-toolie

# Для изменений нажимать кнопку "Перезагрузить" на странице расширений
```

## 🎯 Типы контрибьюций

### 🐛 Исправление ошибок

1. Создать Issue с описанием проблемы
2. Включить: OS, версию браузера, версию расширения
3. Предоставить шаги для воспроизведения
4. Создать PR с исправлением
5. Включить тех, кто может протестировать

### ✨ Новые функции

1. **Обсудить идею** перед начало кодирования
2. Создать Issue с описанием:
   - Проблема, которую решает функция
   - Предложенное решение
   - Альтернативные подходы
3. После одобрения создать PR
4. Включить документацию

### 📚 Документация

1. Обновить соответствующий файл (README, INSTALLATION, и т.д.)
2. Проверить орфографию
3. Убедиться, что примеры работают
4. Включить скриншоты/диаграммы если нужно

### 🎨 Улучшения дизайна

1. Создать Issue с мокапом или описанием
2. Обсудить с командой
3. Реализовать согласно Material Design 3 гайдлайнам
4. Тестировать на светлой и темной темах

## ✅ Чеклист перед отправкой PR

- [ ] Проверен код на ошибки (F12 Console)
- [ ] Тестирование на нескольких браузерах (Chrome, Edge, Brave)
- [ ] Темная тема работает корректно
- [ ] Адаптивность на мобильных устройствах
- [ ] Комментарии добавлены для сложного кода
- [ ] Нет console.log() для продакшена (только для отладки)
- [ ] Git история чистая (логичные commits)
- [ ] PR описание полное и ясное
- [ ] Связан с Issue если это баг-фикс
- [ ] Обновлена документация если нужно

## 💬 Процесс ревью

1. **Автоматические проверки**
   - Синтаксис JavaScript
   - CSS style linting
   - Совместимость браузеров

2. **Ревью кода**
   - Качество кода и лучшие практики
   - Производительность
   - Безопасность
   - Соответствие стилю проекта

3. **Тестирование**
   - Функциональное тестирование
   - Регрессионное тестирование
   - Тестирование производительности

4. **Одобрение и мерж**
   - Минимум 1 одобрение
   - Все проверки должны пройти

## 🔐 Безопасность

### Правила безопасности

1. **API ключи**
   - Никогда не коммитить настоящие ключи
   - Использовать `.env` для локальной разработки
   - Все ключи идут в `.gitignore`

2. **Чувствительные данные**
   - Никогда не логировать чувствительные данные
   - Маскировать пароли в UI
   - Валидировать все входные данные

3. **Инъекции кода**
   - Использовать `textContent` вместо `innerHTML`
   - Избегать `eval()`
   - Санитизировать пользовательский ввод
